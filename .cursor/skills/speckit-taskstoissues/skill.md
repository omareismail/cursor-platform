# Skill: speckit-taskstoissues

**Invocation:** `/speckit-taskstoissues`

---

## Overview

`speckit-taskstoissues` converts the task board from `speckit-tasks` into ready-to-run
`gh` CLI commands that create GitHub Issues with full metadata: title, body, labels, and
milestone. It also generates the milestone creation command and any label creation commands
for labels that do not yet exist in the repository. The resulting issues provide end-to-end
traceability from spec acceptance criteria through to GitHub issue tracking to commit messages.
It detects the GitHub remote URL from `git remote -v` to fill in the `:owner/:repo` values.

---

## Steps

**Step 1 — Read inputs.**

- `memory-bank/progress.md` — active feature task board (the ⬜ Not Started section)
- `specs/features/[feature-slug].md` — acceptance criteria for linking
- `git remote -v` — to extract the GitHub owner and repo name

**Step 2 — Parse remote URL.**

```bash
# Example: git remote -v output
origin  https://github.com/omar-hassan/orient-portal.git (fetch)
# Extracted: owner=omar-hassan, repo=orient-portal
```

**Step 3 — Generate milestone creation command.**

```bash
# Create milestone for the feature
gh api repos/[owner]/[repo]/milestones \
  --method POST \
  --field title="[Feature Name]" \
  --field description="Spec: specs/features/[feature-slug].md" \
  --field due_on="[YYYY-MM-DDT00:00:00Z]"
```

**Step 4 — Generate label creation commands.**

For each label that may not exist yet:

```bash
# Layer labels
gh label create "layer:domain"         --color "7B61FF" --repo [owner]/[repo]
gh label create "layer:application"    --color "0075CA" --repo [owner]/[repo]
gh label create "layer:infrastructure" --color "E4E669" --repo [owner]/[repo]
gh label create "layer:api"            --color "D93F0B" --repo [owner]/[repo]
gh label create "layer:react"          --color "0E8A16" --repo [owner]/[repo]
gh label create "layer:tests"          --color "FBCA04" --repo [owner]/[repo]
# Size labels
gh label create "size:S" --color "C2E0C6" --repo [owner]/[repo]
gh label create "size:M" --color "F9D0C4" --repo [owner]/[repo]
gh label create "size:L" --color "E99695" --repo [owner]/[repo]
```

**Step 5 — Generate one `gh issue create` command per task.**

```bash
gh issue create \
  --title "[LAYER] TASK-001: Create [Entity] aggregate + value objects" \
  --body "## Task Description
Create the [Entity] aggregate root with private setters, factory method, and
invariant checks as defined in the spec.

## Acceptance Criteria (from spec)
- AC-1: [criterion text]
- AC-2: [criterion text]

## Spec Reference
specs/features/[feature-slug].md

## Layer
Domain

## Size Estimate
S (< 2 hours)

## Depends On
None

## Notes
- Follow systemPatterns.md §DomainEntities pattern
- Private setters, no public constructors, Result<T> return from Create()" \
  --label "layer:domain,size:S" \
  --milestone "[Feature Name]" \
  --repo [owner]/[repo]
```

Repeat for every task in the Not Started board, mapping:
- `Domain` → `layer:domain`
- `Application` → `layer:application`
- `Infrastructure` → `layer:infrastructure`
- `API` → `layer:api`
- `React/API`, `React/UI` → `layer:react`
- `Tests` → `layer:tests`
- `S` → `size:S`, `M` → `size:M`, `L` → `size:L`

**Step 6 — Output all commands in a single executable script.**

Wrap all commands in a shell script block:

```bash
#!/bin/bash
# Generated by speckit-taskstoissues
# Feature: [Feature Name]
# Run from repo root after gh auth login

set -e
REPO="[owner]/[repo]"

echo "Creating milestone..."
# [milestone command]

echo "Creating labels (safe to re-run — errors on existing labels are ignored)..."
# [label commands] 2>/dev/null || true

echo "Creating issues..."
# [issue commands]

echo "✅ All issues created."
```

## Example Invocation

**Scenario:** `broker-client-csv-export` task board has 17 tasks. User runs `/speckit-taskstoissues`.

Agent detects remote `https://github.com/omar-hassan/orient-portal.git`,
generates 17 `gh issue create` commands, links each to the relevant acceptance
criteria from the spec, and wraps everything in a runnable `scripts/create-issues-broker-export.sh`.

## Output

- File: `scripts/create-issues-[feature-slug].sh` (executable shell script)
- Terminal: summary of how many issues will be created, plus reminder to `chmod +x` the script
